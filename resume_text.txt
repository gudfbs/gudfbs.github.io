5. 자기소개서 (Q&A 이력서)
-----------------------------------------
[Q] QA로서 본인이 기여하고 집중한 부분과 성과(또는 실패 이후의 개선 사례)를 알고 싶어요.

[A]
제가 QA를 수행할 때 가장 중요하게 생각하는 부분은 단순 기능 검증이 아닌 실제 사용자 흐름 관점에서 서비스 안정성을 확인하는 것입니다.

이셀러스 메인 서비스 QA에서는 상품·주문·배송 흐름 기반 테스트를 수행하며 운영 이슈를 직접 재현하고 원인을 분석해 개발 및 기획과 협업했습니다. 특히 사용자 시나리오 기반 테스트를 통해 오픈마켓 연동 과정에서 발생하는 예외 케이스를 사전에 발견하며 실제 운영 장애를 예방한 경험이 있습니다.

또한 이셀러스Pro 프로젝트에서는 전수 QA 리딩을 담당하며 기능별 테스트 시나리오와 회귀 테스트 기준을 정리했습니다. 반복 검증 항목을 구조화하고 우선순위를 정리하면서 QA 처리 건수가 약 24% 증가하는 결과로 이어졌습니다.

이 경험을 통해 QA는 단순히 오류를 찾는 역할이 아니라, 사용자 경험과 서비스 흐름 전체를 이해하며 리스크를 줄이는 역할이라고 생각하게 되었습니다.

[Q] 빠른 개발/배포 사이클 속에서 QA를 수행해보신 경험이 있다면 작성해 주세요.

[A]
현재 운영 중인 이커머스 솔루션은 오픈마켓 정책 변경과 신규 기능 배포가 자주 발생하는 환경이기 때문에 빠른 개발 및 배포 사이클 속에서 QA를 수행해왔습니다.

모든 기능을 동일한 수준으로 검증하기 어려운 상황에서는 실제 사용자 영향도가 높은 영역을 우선적으로 확인하는 방식으로 QA를 진행했습니다. 특히 상품 등록, 주문 수집, 배송 처리와 같이 운영에 직접 영향을 주는 기능을 우선 검증 대상으로 분류하고 사용자 시나리오 기반으로 핵심 흐름을 빠르게 확인하는 데 집중했습니다.

또한 기존 운영 이슈와 반복 발생 장애 이력을 참고해 회귀 테스트 범위를 조정하며 제한된 일정 안에서도 안정적인 오픈 QA를 수행했습니다.

[Q] 다양한 기술 스택을 활용 했다면, 해당 기술을 실제 QA에 어떻게 활용했는지 작성해 주세요.

[A]
업무에서는 JIRA와 Confluence를 활용해 테스트 결과와 이슈 재현 절차를 문서화하고 있으며, 운영 이슈 발생 시 영향 범위와 재현 과정을 정리해 개발 및 기획과 빠르게 협업할 수 있도록 노력했습니다.

또한 GitHub 및 VSCode 환경을 활용해 Playwright 기반 테스트 자동화를 개인적으로 학습하고 있습니다. 반복적으로 수행되는 웹 시나리오 테스트를 자동화하는 과정을 직접 구성해보며 테스트 효율화 관점에서 접근하고 있습니다.

특히 자동화 도구 자체보다 어떤 테스트를 자동화해야 효율적인지 판단하는 것이 중요하다고 생각하며, 반복 빈도가 높고 결과 검증이 명확한 영역 중심으로 자동화를 고려하고 있습니다.
